## Overview

Sift's AI Disaster Discovery Agent is an autonomous system that continuously monitors multiple global data sources to discover and validate disaster events. Built with OpenAI's function-calling capabilities, it runs on a scheduled interval to fetch, analyze, and save verified disaster alerts.

## How It Works

The agent operates on an autonomous cycle:

1. **Data Collection**: Queries all configured data sources in parallel
2. **Event Analysis**: Uses AI to evaluate significance and severity
3. **Deduplication**: Filters out duplicate events within the same geographic area
4. **Alert Creation**: Saves validated events and broadcasts to connected clients

<Info>
The agent runs automatically every few minutes (configurable via `AGENT_INTERVAL_MINUTES`) and performs a full scan of all data sources on each run.
</Info>

## Deduplication Logic

To prevent duplicate alerts, the agent uses geographic proximity checking:

```python
DEDUP_RADIUS_KM = 50.0  # Only alerts of the same type within this radius are duplicates

def _is_duplicate(self, alert_type: str, lat: float, lng: float, hours: int = 2) -> bool:
    since = datetime.now(timezone.utc) - timedelta(hours=hours)
    for alert in storage.get_all_alerts():
        # Skip if different type or inactive
        if alert.get("type") != alert_type or not alert.get("active", True):
            continue
        # Skip if older than time window
        created = datetime.fromisoformat(alert["created_at"])
        if created < since:
            continue
        # Check distance using Haversine formula
        if _distance_km(lat, lng, alert["lat"], alert["lng"]) < DEDUP_RADIUS_KM:
            return True
    return False
```

**Key insight**: Different disaster types (e.g., earthquake vs tsunami) in the same area are both saved, since they may be related but distinct events.

**Implementation**: `server/app/services/disaster_agent.py:479-498`

## Save Criteria

The agent only saves events that meet ALL criteria:

<Steps>
  <Step title="Real Event">
    From an official API or confirmed news source
  </Step>
  <Step title="Current">
    Occurred within the last 6 hours
  </Step>
  <Step title="Significant">
    Meets severity threshold (e.g., M4.5+ earthquakes, Severe/Extreme weather)
  </Step>
  <Step title="Not Duplicate">
    No similar event of the same type within 50km in the past 2 hours
  </Step>
</Steps>

## Execution Flow

The agent uses an iterative execution loop with a 25-iteration limit:

<CodeGroup>
```python Agent Loop
for iteration in range(25):
    response = client.chat.completions.create(
        model="Qwen3-30B-A3B",
        messages=messages,
        tools=TOOLS,
        tool_choice="auto",
        max_tokens=4096,
        temperature=0.1,
    )
    
    msg = response.choices[0].message
    
    # If no tool calls, agent is done
    if not msg.tool_calls:
        break
    
    # Execute each tool call
    for tool_call in msg.tool_calls:
        result = await self._execute_tool(tool_call.function.name, args)
        # Add result to conversation
        messages.append({"role": "tool", "content": json.dumps(result)})
```

## Scheduling

The agent is scheduled using APScheduler on server startup:

```python
scheduler = AsyncIOScheduler(timezone="UTC")
scheduler.add_job(
    agent.run,
    trigger="interval",
    minutes=interval,  # From config: AGENT_INTERVAL_MINUTES
    next_run_time=datetime.now(timezone.utc),  # Run immediately on startup
    id="disaster_agent",
    misfire_grace_time=60,
)
scheduler.start()
```
