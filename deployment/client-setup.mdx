This guide covers setting up the Sift React Native client for Android and iOS devices.

## Prerequisites

### Required Software

- **Node.js** 18 or higher
- **npm** (comes with Node.js)
- **React Native CLI**

### Platform-Specific Requirements

<Tabs>
  <Tab title="Android">
    - **Android Studio** (latest stable version)
    - **Android SDK** (API level 21 or higher)
    - **JDK** 11 or higher
    - **Android device** or emulator with Bluetooth support
    - **USB debugging** enabled on physical device
  </Tab>
  
  <Tab title="iOS">
    - **macOS** (required for iOS development)
    - **Xcode** 12 or higher
    - **CocoaPods** installed
    - **iOS device** (11.0+) or simulator
    - **Apple Developer account** (for physical device deployment)
  </Tab>
</Tabs>

## Installation

<Steps>
  <Step title="Navigate to client directory">
    ```bash
    cd Client
    ```
  </Step>

  <Step title="Install dependencies">
    Install all npm packages:
    
    ```bash
    npm install
    ```
    
    This installs:
    - React Native 0.76.5
    - React Native BLE PLX - Bluetooth Low Energy scanning
    - React Native Multi BLE Peripheral - BLE advertising
    - AsyncStorage - Local data persistence
    - Axios - HTTP client
    - Additional utilities (uuid, buffer, etc.)
  </Step>

  <Step title="Configure server URL">
    Update the server URL to point to your backend. Edit `src/config/constants.js`:
    
    ```javascript
    export const APP_CONFIG = {
      CENTRAL_SERVER_URL: 'http://YOUR_SERVER_IP:8001',
      VERSION: '1.0.0',
      DEBUG_MODE: __DEV__ ?? false,
    };
    ```
    
    **Examples:**
    - Local development: `http://192.168.1.100:8000` (use your machine's LAN IP)
    - Remote server: `http://165.245.139.104:8001`
    - ngrok tunnel: `https://your-tunnel.ngrok-free.app`
    
    <Warning>
      Do not use `localhost` or `127.0.0.1` when testing on a physical device - these refer to the device itself, not your development machine.
    </Warning>
  </Step>

  <Step title="Platform-specific setup">
    <Tabs>
      <Tab title="Android">
        **Install Android SDK and tools:**
        
        1. Open Android Studio
        2. Go to Tools → SDK Manager
        3. Install Android SDK Platform 21 or higher
        4. Install Android SDK Build-Tools
        
        **Configure environment variables:**
        
        Add to your `~/.bashrc` or `~/.zshrc`:
        ```bash
        export ANDROID_HOME=$HOME/Android/Sdk
        export PATH=$PATH:$ANDROID_HOME/emulator
        export PATH=$PATH:$ANDROID_HOME/platform-tools
        ```
        
        **Enable USB debugging on device:**
        
        1. Go to Settings → About phone
        2. Tap "Build number" 7 times to enable Developer options
        3. Go to Settings → Developer options
        4. Enable "USB debugging"
      </Tab>
      
      <Tab title="iOS">
        **Install iOS dependencies:**
        
        ```bash
        cd ios
        pod install
        cd ..
        ```
        
        **Configure signing:**
        
        1. Open `ios/sift.xcworkspace` in Xcode
        2. Select the project in the navigator
        3. Go to Signing & Capabilities
        4. Select your development team
        5. Xcode will automatically create a provisioning profile
      </Tab>
    </Tabs>
  </Step>

  <Step title="Start Metro bundler">
    Start the Metro JavaScript bundler:
    
    ```bash
    npm start
    ```
    
    Keep this terminal running. Metro will bundle your JavaScript code and serve it to the app.
  </Step>

  <Step title="Run the app">
    **Open a new terminal** (keep Metro running in the first terminal):
    
    <Tabs>
      <Tab title="Android">
        **Connect device or start emulator**, then run:
        
        ```bash
        cd Client
        npm run android
        ```
        
        **To verify connected devices:**
        ```bash
        npm run android:devices
        # or directly:
        adb devices
        ```
        
        **Run on specific device:**
        ```bash
        npm run android:device
        ```
      </Tab>
      
      <Tab title="iOS">
        **Start iOS simulator or connect device**, then run:
        
        ```bash
        cd Client
        npm run ios
        ```
        
        **Run on specific simulator:**
        ```bash
        npx react-native run-ios --simulator="iPhone 14 Pro"
        ```
        
        **Run on physical device:**
        ```bash
        npx react-native run-ios --device
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Grant permissions">
    When the app launches, grant the required permissions:
    
    - **Location** - Required for Bluetooth scanning
    - **Bluetooth** - Required for BLE mesh networking
    - **Notifications** - For alert push notifications (optional)
    
    <Note>
      On Android 12+, you need "Nearby devices" permission for Bluetooth.
    </Note>
  </Step>
</Steps>

## Development Workflow

### Metro Commands

While Metro is running, press:
- `r` - Reload app
- `d` - Open developer menu
- `i` - Run on iOS
- `a` - Run on Android

### Enable Debug Mode

**On Android:**
1. Shake device or press `Cmd+M` (Mac) / `Ctrl+M` (Windows/Linux)
2. Select "Debug"

**On iOS:**
1. Shake device or press `Cmd+D`
2. Select "Debug"

### View Logs

<Tabs>
  <Tab title="Android">
    ```bash
    npx react-native log-android
    ```
    
    Or use adb directly:
    ```bash
    adb logcat
    ```
  </Tab>
  
  <Tab title="iOS">
    ```bash
    npx react-native log-ios
    ```
  </Tab>
</Tabs>

## Troubleshooting

### Metro Bundler Port Already in Use

**Error:** `Port 8081 already in use`

**Solution:**
```bash
lsof -ti:8081 | xargs kill -9
npm start
```

### Build Failures

**Android:**
```bash
cd android
./gradlew clean
cd ..
npm run android
```

**iOS:**
```bash
cd ios
rm -rf build
pod deintegrate
pod install
cd ..
npm run ios
```

### Cannot Connect to Development Server

**On physical Android device:**

1. Ensure device and computer are on the same network
2. Enable port forwarding:
   ```bash
   adb reverse tcp:8081 tcp:8081
   ```

### Bluetooth Permissions Denied

**Android:**
- Check app permissions in Settings → Apps → Sift → Permissions
- Ensure Location is enabled (required for BLE scanning)

**iOS:**
- Go to Settings → Sift → Allow Bluetooth
- Ensure Location Services are enabled

### App Cannot Reach Server

**Checklist:**
1. Server is running and accessible
2. `CENTRAL_SERVER_URL` in `constants.js` is correct
3. Using LAN IP (not localhost) for physical devices
4. Server firewall allows the port
5. Device and server can ping each other

**Test connection:**
```bash
# From your computer
curl http://YOUR_SERVER_IP:8001/health
```
