## What is Sift?

Sift is a full-stack disaster management application that aggregates alerts from multiple authoritative sources, enriches them with AI-powered analysis, and delivers them to mobile clients over the internet or via Bluetooth when connectivity is limited.

## Key Features

### Real-Time Disaster Alerts

Sift aggregates alerts from multiple authoritative sources:

- **USGS** — Earthquakes (M4.5+)
- **NOAA** — Severe weather alerts (floods, storms, hurricanes)
- **NASA EONET** — Wildfires, volcanoes, severe storms
- **GDACS** — Global disaster alerts (Red/Orange level)
- **Web Search** — Recent disaster news via Tavily

### AI-Powered Discovery Agent

An autonomous agent runs every 20 minutes to:

<Steps>
  <Step title="Fetch from all sources">
    Queries USGS, NOAA, NASA EONET, and GDACS APIs for recent events
  </Step>
  <Step title="Deduplicate events">
    Filters out duplicate alerts within 50km of same type
  </Step>
  <Step title="Assess severity">
    Evaluates each event and assigns severity levels (low, medium, high, critical)
  </Step>
  <Step title="Save & broadcast">
    Stores qualifying alerts and pushes them to all connected clients
  </Step>
</Steps>

```python
# From disaster_agent.py - The agent's workflow
for iteration in range(25):
    response = client.chat.completions.create(
        model="Qwen3-30B-A3B",
        messages=messages,
        tools=TOOLS,
        tool_choice="auto",
        max_tokens=4096,
        temperature=0.1,
    )
    
    # Agent calls tools: fetch_usgs_earthquakes, fetch_noaa_alerts,
    # fetch_nasa_eonet, fetch_gdacs_alerts, search_web, save_disaster_alert
```

### Real-Time Push Notifications

WebSocket connections ensure connected clients receive new alerts instantly:

```javascript
// From App.js - WebSocket setup
websocketService.connect(APP_CONFIG.CENTRAL_SERVER_URL, deviceId);
websocketService.setOnAlert(async (alert) => {
  await alertService.processAlert(alert, ALERT_SOURCE_SERVER);
});
```

### BLE Mesh for Offline Sharing

When internet connectivity is limited or unavailable, Sift devices form a Bluetooth Low Energy mesh network:

- Phones **advertise** as BLE peripherals
- **Scan** for other Sift devices nearby
- **Share alerts** over Bluetooth
- **In-app chat** for peer-to-peer messaging

<Note>
The BLE mesh is critical for disaster scenarios where cellular networks may be overloaded or down.
</Note>

```javascript
// From App.js - BLE mesh initialization
await blePeripheralService.start(advertiseName, async (data) => {
  if (data && data._type === 'chat') {
    await onChatMessage(data);
    return;
  }
  await alertService.processAlert(data, ALERT_SOURCE_BLUETOOTH);
});

bluetoothService.startScanning((device) => {
  if (!bluetoothService.isDeviceConnected(device.id)) {
    bluetoothService.connectToDevice(device);
  }
});
```

## Alert Types & Severity

Sift supports the following alert types:

- **Earthquake** — Seismic events from USGS
- **Flood** — Flash floods, river flooding
- **Fire** — Wildfires, chemical fires
- **Storm** — Severe weather, hurricanes, tornadoes
- **Chemical** — Industrial accidents, spills
- **Tsunami** — Ocean-based threats
- **Medical** — Public health emergencies
- **Infrastructure** — Bridge collapses, power outages
- **Other** — Miscellaneous disasters

Each alert is assigned a severity level:
